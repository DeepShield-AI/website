DeepTrace

DeepTrace 是一个非侵入式、基于事务分析进行请求关联的分布式追踪框架，旨在解决大规模微服务系统在生产环境中监控和诊断的挑战。

它无需修改应用代码，利用主机端的事务分析机制来实现高效且准确的端到端请求追踪。

其核心目标包括多协议兼容性、精确追踪、高效操作和低性能开销。

DeepTrace 通过部署在主机上的 Agent 和中心 Server 协同工作，已在数十家公司的生产环境中成功部署，用于故障诊断、资源优化等任务，并在高并发下保持超过 95% 的追踪准确率。

跨协议追踪

DeepTrace 实现了非侵入式的跨协议 Span 构建。 它利用 eBPF 技术在系统调用层非侵入式地捕获原始网络数据（PID、Socket 五元组、时间戳、消息内容）。 基于对 RFC 和开源实现的分析，DeepTrace 设计了包含协议类型推断和内容分区的协议模板（支持超过 20 种主流应用层协议）。协议类型推断通过匹配请求消息中特定位置的字段值范围来完成。内容分区则利用协议内置的长度字段进行“跳跃式解析”，高效准确地分割请求边界（对于无长度字段的协议则进行全解析）。

高精度的请求关联

DeepTrace 的核心创新在于其基于事务的 Span 关联机制，以解决传统 非侵入式方案在高并发下的追踪准确率局限。它通过分析有状态的请求内容来推断请求之间的因果关系，包括 API 调用关系、请求内容中的事务字段。同时，它综合多维 Span 指标（如起止时间延迟、请求大小、持续时间）的概率分布，从而准确的多维度评估请求之间的父子关系概率。即使在高并发小依然能保持高精度的追踪准确率。

轻量级按需采集

为了显著降低海量 Span 的传输和存储开销，DeepTrace 采用查询驱动的按需追踪组装策略。Agent 在内存中临时存储 Span 时，并构建双重索引：1) 为每种标签类型（如主机名、服务名）建立倒排索引；2) 为每种度量指标（如持续时间）构建直方图（按值分桶）来降低缓存开销。这种设计避免了相同标签和相似度量值的冗余存储。当操作员通过 API 提交查询时，Server 将查询下发给所有 Agent。Agent 利用标签索引和度量直方图快速筛选出相关的 Span ，从而避免收集所有无关 Span 的开销。
